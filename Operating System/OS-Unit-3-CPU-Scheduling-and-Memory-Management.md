# OS Unit 3 - CPU Scheduling and Memory Management

---

## 1. Shortest Remaining Time First (SRTF)

SRTF is the preemptive version of Shortest Job First (SJF).

* CPU is allocated to the process with the smallest remaining burst time
* If a new process arrives with smaller burst time → preemption occurs

---

### Key Points:

* Minimizes average waiting time
* Preemptive in nature
* Difficult to implement due to continuous checking

---

## 2. System Calls

System calls provide an interface between user programs and the operating system.

---

### Types of System Calls:

1. Process Control

   * fork(), exit(), wait()

2. File Management

   * open(), read(), write(), close()

3. Device Management

   * request device, release device

4. Information Maintenance

   * get time, system data

5. Communication

   * send(), receive()

---

## 3. Dispatcher

Dispatcher is the module that gives control of CPU to the process selected by the scheduler.

---

### Functions:

* Context switching
* Switching to user mode
* Jumping to correct instruction

---

### Dispatcher Latency:

Time taken to stop one process and start another

---

## 4. Memory Management

Memory management handles movement of processes between RAM and disk.

---

### Goals:

* Maximize memory utilization
* Improve system performance

---

## 5. Contiguous Memory Allocation

* Process stored in single block

### Advantages:

* Simple
* Fast

### Disadvantages:

* External fragmentation

---

## 6. Non-Contiguous Memory Allocation

* Process divided into parts

### Types:

* Paging
* Segmentation

---

## 7. Fragmentation

---

### Internal Fragmentation

* Memory wasted inside allocated block

---

### External Fragmentation

* Free memory scattered in small blocks

---

### Solution:

* Compaction
* Paging

---

## 8. Paging

Paging divides memory into:

* Pages (logical memory)
* Frames (physical memory)

---

### Address Translation:

* Page number + offset

---

### Page Table:

* Maps pages to frames

---

### Advantage:

* Eliminates external fragmentation

### Disadvantage:

* Internal fragmentation

---

## 9. Segmentation

* Memory divided into variable size segments

---

### Features:

* Each segment has base + limit
* Matches user view

---

### Advantage:

* Logical structure

### Disadvantage:

* External fragmentation

---

## 10. Paging vs Segmentation

| Feature       | Paging      | Segmentation  |
| ------------- | ----------- | ------------- |
| Size          | Fixed       | Variable      |
| Fragmentation | Internal    | External      |
| Table         | Page table  | Segment table |
| View          | System view | User view     |

---

## 11. Page Replacement Algorithms

---

### Page Fault

Occurs when required page is not in memory

---

### FIFO (First In First Out)

* Oldest page replaced
* Simple
* Suffers from Belady’s anomaly

---

### LRU (Least Recently Used)

* Replaces least recently used page
* Better performance

---

### Optimal Algorithm

* Replaces page not used for longest future time
* Best but impractical

---

## 12. Virtual Memory

Virtual memory allows execution of programs larger than physical memory.

---

### Concept:

* Only required pages loaded into RAM
* Rest stored on disk

---

### Advantages:

* Better memory utilization
* Supports multiprogramming

---

## 13. Demand Paging

* Pages loaded only when needed

---

### Steps:

1. CPU requests page
2. Page fault occurs
3. OS loads page from disk
4. Page table updated
5. Execution resumes

---

## 14. Thrashing

Thrashing occurs when system spends more time swapping pages than executing.

---

### Causes:

* Insufficient memory
* High multiprogramming

---

### Solution:

* Reduce processes
* Increase memory

---

## 15. Logical vs Physical Address

| Feature      | Logical Address | Physical Address |
| ------------ | --------------- | ---------------- |
| Generated by | CPU             | MMU              |
| Visibility   | User            | Not visible      |
| Nature       | Virtual         | Real             |

---

## 16. Swapping

Swapping is moving processes between RAM and disk.

---

### Steps:

* Swap out → move to disk
* Swap in → bring back to memory

---

### Purpose:

* Improve memory utilization
* Allow more processes

---

## Summary

* SRTF is preemptive scheduling
* System calls connect user and OS
* Paging & segmentation manage memory
* Page replacement improves performance
* Virtual memory enables large programs
* Thrashing reduces performance
* Swapping helps manage memory
